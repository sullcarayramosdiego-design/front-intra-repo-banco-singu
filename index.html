Presentación Ejecutiva

Prueba Técnica - Arquitectura y Desarrollo Full-Stack

Refactorización del Pipeline de datos, modernización arquitectónica y construcción de un Dashboard Gerencial dinámico para el Banco Singular.

1. Contexto y Objetivos

Entendiendo el desafío del Banco Singular

El Desafío del Negocio

  • Punto de Partida: El banco genera exportaciones estáticas en formato CSV desde su core transaccional.
  • El Objetivo: Construir un pipeline automatizado, con integridad referencial, para alimentar un data warehouse moderno y reportes directivos.
  • Requerimientos Clave:
    - Cartera activa: Saldo consolidado interactivo por tipo de producto y región.
    - Seguridad SSO: Integración estricta con Keycloak.
    - Limpieza de Data: Normalización de catálogos y atributos (ej. "FISICA" a "Natural").
  • Visión: Transformar datos aislados en inteligencia de negocio centralizada, segura y de alta disponibilidad con visualizaciones avanzadas.

Enfoque Arquitectónico

Rechazo al Monolito

Implementamos soluciones programáticas y ágiles (Python + Node.js) en lugar de herramientas pesadas. Versionamos toda la lógica en Git, lo que permite detectar rápidamente errores de Integridad Referencial en el código ETL y refactorizar a demanda.

Contenedores y Microservicios

Adoptamos una arquitectura orientada a microservicios administrada bajo Dokploy. Esto garantiza que la base de datos (MySQL), el backend, el IdP (Keycloak) y el frontend corran de forma aislada, facilitando flujos de integración continua.

Stack Tecnológico Elegido

ETL & Datos

Python (Pandas + SQLAlchemy). Detectamos y solucionamos problemas de integridad insertando las llaves correctas cruzando tablas de dimensiones y hechos en memoria antes de la carga a MySQL.

API Backend

Node.js con Express estructurado en capas. Elimina tokens "mock" para validar estrictamente la llave pública (JWKS) desde Keycloak, ofreciendo endpoints REST ultrarrápidos y seguros.

Dashboard UI

Next.js (App Router) con Recharts y Tailwind CSS. Se introdujeron visualizaciones de "Barras Apiladas" y Sidebars interactivos, mejorando radicalmente la UX sobre la simple presentación de tablas.

2. Trabajo Reciente y Resolución de Problemas

Estrategia de ejecución técnica aplicada

Fases de Despliegue (Flujo Dokploy)

Fase 1: Infra

Despliegue de MySQL y Keycloak en contenedores aislados mediante Dokploy.

Fase 2: ETL

Procesamiento automatizado de los archivos CSV utilizando Python y Pandas.

Fase 3: Backend

Desarrollo de API Node.js con protección OIDC y mapeo de consultas SQL.

Fase 4: Front

Construcción del dashboard directivo interactivo usando Next.js y Recharts.

Eficiencia del Pipeline de Datos

100%
Automatizado e Íntegro

Del Archivo Plano al Relacional

El script en Python se ajustó para asegurar la Integridad Referencial. Al insertar datos en fact_transacciones, el ETL ahora cruza en memoria las cuentas para inyectar dinámicamente las llaves primarias correctas (ej. cliente_id), evitando errores de base de datos críticos (Error 1364) y garantizando que el modelo estrella esté siempre consistente.

Seguridad Cero Mocks (Zero Trust)

  • Proveedor de Identidad: Autenticación completamente centralizada en Keycloak usando OIDC.
  • Eliminación de Mocks: Se purgó del Backend todo código de simulación (bypass) o claves secretas de desarrollo locales.
  • Validación Estricta: Node.js ahora valida tokens comprobando estrictamente el emisor y el destinatario contra el JWKS (JSON Web Key Set) de Keycloak en producción.
  • Prevención de Fugas: Sin riesgo de accesos accidentales o tokens caducados siendo procesados.

3. Resultados y Entregables

Dashboard Interactivo y Analítica Avanzada

Informe de Mejoras en la UI/UX

  • Gráficos Avanzados (Barras Apiladas): Se reemplazó la tabla plana de Cartera Activa por un componente interactivo de Recharts que segmenta el saldo consolidado por región y lo divide por tipo de producto financiero.
  • Layout de Filtros: Se reestructuró la página principal para incluir una barra lateral (Sidebar) dedicada a los filtros de segmentación comercial y localización geográfica.
  • Limpieza de Catálogos Front-End: El dashboard ahora mapea inteligentemente la data cruda del ETL. Por ejemplo, los clientes reportados como "FISICA" o "MORAL" se consolidan correctamente bajo los KPIs visuales de "Personas Naturales" y "Jurídicas", corrigiendo errores de métricas en 0.
  • Resolución de Tipado: Se aplicó "type safety" (TypeScript) a las propiedades avanzadas de gráficos, previniendo fallos en compilación o renderizado durante los pasos a producción.
